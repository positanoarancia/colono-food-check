# AGENTS.md

## 작업 시작 규칙
1. AGENTS.md를 먼저 읽고 규칙을 따른다.
2. README.md를 읽고 프로젝트 맥락을 이해한다.
3. STATUS.md를 읽고 현재 상태를 파악한다.
4. TASK.md를 기준으로 위에서부터 순서대로 작업한다.

---

## 작업 원칙
- 각 큰 작업 시작 전 3~7줄의 짧은 실행 계획을 작성한다.
- MVP 우선으로 구현하고 과설계하지 않는다.
- 변경 범위는 최소화한다.
- 기존 구조와 스타일을 최대한 유지한다.
- 모호한 부분은 README.md와 AGENTS.md 기준으로 판단한다.

---

## 구현 규칙
- 불필요한 라이브러리 추가 금지
- 가능한 단순한 구조 유지
- 기능 단위로 작게 구현
- UI/로직을 명확히 분리

---

## 🔥 멀티 역할 리뷰 루프 (파일 기반)

각 작업 완료 후 반드시 아래 순서를 수행한다:

---

### 1. Planner 리뷰 (PLANNER.md 기준)
- PLANNER.md를 기준으로 기획 검토 수행
- 사용자 가치, 기능 필요성, 흐름, SEO 관점 평가

출력:
- 개선 제안 목록
- 왜 필요한지
- 적용 우선순위

---

### 2. Designer 리뷰 (DESIGNER.md 기준)
- DESIGNER.md를 기준으로 UI/UX 검토 수행
- 모바일 UX, 버튼 흐름, 시각 구조 개선

출력:
- UI 문제점
- UX 개선안
- 적용 우선순위

---

### 3. 개선 반영
- Planner + Designer 제안 중 중요한 항목만 반영
- MVP 범위 유지
- 변경 범위 최소화

---

### 4. 코드 검증
1. 코드 self-review
2. lint / typecheck / test 실행
3. 실패 시 수정 후 재검증

---

### 5. TASK 업데이트
- 완료 항목 체크
- 필요 시 새로운 작업 추가

---

### 6. STATUS 업데이트 (중요)
아래 내용 반드시 반영:

- 현재 상태 업데이트
- 최근 작업 기록 추가
- 현재 이슈 업데이트
- 다음 작업 정의

---

## 보고 형식
각 작업 완료 후 반드시 아래 형식으로 보고:

- 무엇을 구현했는지
- 어떤 파일을 수정했는지
- 어떤 검증을 통과했는지
- Planner 개선 제안 요약
- Designer 개선 제안 요약
- 현재 상태 요약
- 남은 리스크

---

## 품질 기준
- 실제 실행 가능한 상태 유지
- 에러 없이 동작
- 기본 사용자 흐름 정상
- 최소 UX 품질 확보

---

## 금지 사항
- 과도한 리팩토링
- 요구되지 않은 기능 추가
- 전체 구조 변경
- 불필요한 최적화
- 디자인 때문에 복잡도 증가

---

## 핵심 목표
TASK.md 기반으로 작업을 완료하고,
STATUS.md를 통해 상태를 유지하며,
PLANNER.md와 DESIGNER.md를 통해 지속적으로 품질을 개선한다.
