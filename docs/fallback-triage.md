# fallback 검색어 분류

## 목적

이 문서는 `SearchLog`에서 `matchedType = fallback` 또는 `confidenceGrade = C`로 떨어진 검색어를 운영 관점에서 빠르게 분류하기 위한 기준과 현재 후보 목록을 정리한다.

## 현재 기준

- `alias`: 기존 대표 food와 사실상 같은 음식
- `food_group`: 세부 종류는 다양하지만 같은 카테고리로 안내해도 안전한 음식
- `new_food`: 검색량이 높거나 설명 품질 때문에 독립 food가 필요한 음식

## 운영 메모

- 초기 단계에서는 `alias`로 해결 가능한 검색어를 먼저 닫는 것이 가장 비용 대비 효과가 크다.
- `food_group` 흡수는 자동 분류 로직이 없으므로, 우선 운영 문서와 수동 점검 기준으로 관리한다.
- `new_food`는 실제 검색량이 2회 이상 반복되거나, 사용자 문의가 많은 경우 seed에 우선 편입한다.
